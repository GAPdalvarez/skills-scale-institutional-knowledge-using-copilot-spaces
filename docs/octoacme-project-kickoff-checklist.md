# OctoAcme — Project Kickoff Checklist

## Purpose
Provide a lightweight kickoff template aligned to the Initiation and Planning guides so roles, decisions, and artifacts are explicit.

## When to use
After initiation approval (go/no-go) and before detailed sprint/iteration execution.

## Required Inputs (from Initiation)
- [ ] Approved Project One-pager (problem, goal, success metrics)
- [ ] Stakeholder list and communication plan
- [ ] Initial risk list and major dependencies
- [ ] Proposed team / roles and rough timeline

## Required Attendees
- [ ] Project Manager (PM)
- [ ] Product Manager (PdM / Product Lead)
- [ ] Developer representative (engineering lead or delegate)
- [ ] QA/Testing representative
- [ ] UX Designer
- [ ] DevOps / Platform Engineer
- [ ] Security Lead (or delegate)
- [ ] Customer Support Lead
- [ ] Stakeholder representative(s)

## Optional / As Needed Attendees
- [ ] Executive Sponsor (required for high-risk/high-impact initiatives)
- [ ] External Stakeholder representative (partner/customer/regulatory)

## Kickoff Agenda
- [ ] 1) Problem statement, objective, and success metrics review
- [ ] 2) Scope boundaries, assumptions, and constraints
- [ ] 3) Backlog framing, milestone map, and Definition of Done alignment
- [ ] 4) Risks, dependencies, and escalation path confirmation
- [ ] 5) Release approach, quality expectations, and communication cadence
- [ ] 6) Decisions, owners, and next-step commitments

## Required Decisions
- [ ] Confirm go-forward scope for planning
- [ ] Confirm role ownership for backlog, risk register, release readiness, and retro tracking
- [ ] Confirm communication cadence (standups, weekly sync, stakeholder updates)
- [ ] Confirm escalation path (team triage -> PM -> Product Lead -> Sponsor)

## Required Outputs / Artifacts
- [ ] Meeting notes with decisions and open questions
- [ ] Updated backlog seed items with acceptance criteria
- [ ] Draft Definition of Done and QA approach
- [ ] Updated risk register with owners and mitigations
- [ ] Milestone/release timeline draft
- [ ] Action log with owner and due date for each follow-up

## Follow-up (within 48 hours)
- [ ] Publish kickoff notes to the project repo (`docs/` or project README)
- [ ] Confirm owners accepted assigned actions
- [ ] Schedule first delivery sync and milestone checkpoints
- [ ] Share summary with all attendees and stakeholders
